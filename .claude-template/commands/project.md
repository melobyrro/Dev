# Project Command

Display project overview and key information.

## Instructions

Provide a concise project overview including:

1. **Project Identification**:
   - Project name
   - Project type (web app, infrastructure, library, etc.)
   - Primary technologies/stack

2. **Project Purpose**:
   - Brief description of what the project does
   - Main goals and objectives
   - Target users or use cases

3. **Architecture**:
   - Key components and structure
   - Important directories or files
   - External dependencies or services

4. **Development Setup**:
   - How to get started with development
   - Key commands for building, testing, running
   - Environment requirements

5. **Claude Code Integration**:
   - Custom skills available for this project
   - Custom agents or slash commands
   - Recommended workflows for common tasks

6. **Resources**:
   - Link to CLAUDE.md for detailed context (may include architecture)
   - Link to README or other documentation
   - Link to project-specific ARCHITECTURE.md if it exists separately

If CLAUDE.md exists, extract this information from it. Otherwise, analyze the project structure to provide this overview.

Format clearly using markdown with appropriate sections and bullet points.
